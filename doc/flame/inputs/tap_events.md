# События касания

```{note}
В этом документе описан новый API событий. Старый (устаревший) подход,
который всё ещё поддерживается, описан в разделе [](gesture_input.md).
```

**События касания** — один из самых простых способов взаимодействия с игрой Flame. Эти события возникают, когда пользователь касается экрана пальцем, щёлкает мышью или использует стилус. Касание может быть «долгим», но в процессе палец не должен двигаться. Таким образом, касание экрана с последующим перемещением пальца и отпусканием является не касанием, а перетаскиванием. Аналогично, щелчок кнопкой мыши при её движении также будет зарегистрирован как перетаскивание.

Несколько событий касания могут происходить одновременно, особенно если пользователь использует несколько пальцев. Такие случаи корректно обрабатываются Flame, и вы даже можете отслеживать события, используя их свойство `pointerId`.

Для компонентов, которые должны реагировать на касания, добавьте примесь `TapCallbacks`.

- Эта примесь добавляет четыре переопределяемых метода: `onTapDown`, `onTapUp`, `onTapCancel` и `onLongTapDown`. По умолчанию эти методы ничего не делают — для выполнения каких-либо действий их необходимо переопределить.
- Кроме того, компонент должен реализовывать метод `containsLocalPoint()` (уже реализован в `PositionComponent`, поэтому в большинстве случаев ничего делать не нужно). Этот метод позволяет Flame определить, произошло ли событие внутри компонента или нет.

```dart
class MyComponent extends PositionComponent with TapCallbacks {
  MyComponent() : super(size: Vector2(80, 60));

  @override
  void onTapUp(TapUpEvent event) {
    // Выполнить действие в ответ на касание
  }
}
```


## Анатомия касания


### onTapDown

Каждое касание начинается с события «нажатие касания», которое вы получаете через обработчик `void onTapDown(TapDownEvent)`. Событие доставляется первому компоненту в точке касания, имеющему примесь `TapCallbacks`. Обычно после этого распространение события прекращается. Однако вы можете принудительно передать событие нижележащим компонентам, установив `event.continuePropagation` в true.

Объект `TapDownEvent`, передаваемый в обработчик, содержит доступную информацию о событии. Например, `event.localPosition` содержит координаты события в локальной системе координат текущего компонента, а `event.canvasPosition` — в системе координат всего игрового холста.

Каждый компонент, получивший `onTapDown`, в конечном счёте получит либо `onTapUp`, либо `onTapCancel` с тем же `pointerId`.


### onLongTapDown

Если пользователь удерживает палец нажатым некоторое время, срабатывает «долгое касание». Это событие вызывает обработчик `void onLongTapDown(TapDownEvent)` у тех компонентов, которые ранее получили `onTapDown`.

По умолчанию `.longTapDelay` составляет 300 миллисекунд, что может отличаться от системного значения. Вы можете изменить это значение, установив `TapConfig.longTapDelay`. Это также может быть полезно для специальных возможностей доступности.


### onTapUp

Это событие указывает на успешное завершение последовательности касания. Оно гарантированно доставляется только тем компонентам, которые ранее получили `onTapDown` с тем же идентификатором указателя.

Объект `TapUpEvent`, передаваемый в обработчик, содержит информацию о событии, включая координаты (т.е. где пользователь касался экрана непосредственно перед поднятием пальца) и `pointerId`.

Обратите внимание, что координаты устройства для события «касание вверх» будут такими же (или очень близкими) к координатам соответствующего «касания вниз». Однако того же нельзя сказать о локальных координатах. Если компонент, по которому вы касаетесь, перемещается (что часто бывает в играх), вы можете заметить, что локальные координаты «касания вверх» существенно отличаются от локальных координат «касания вниз».

В крайнем случае, когда компонент отодвигается от точки касания, событие `onTapUp` не будет сгенерировано вовсе: оно будет заменено на `onTapCancel`. Однако обратите внимание, что `onTapCancel` в этом случае будет сгенерирован в момент, когда пользователь поднимет или переместит палец, а не в момент, когда компонент уйдёт из-под точки касания.


### onTapCancel

Это событие возникает, когда касание не может завершиться. Чаще всего это происходит, когда пользователь перемещает палец, превращая жест из «касания» в «перетаскивание». Реже это случается, когда компонент, по которому совершается касание, уходит из-под пальца пользователя. Ещё более редко `onTapCancel` возникает, когда другой виджет перекрывает игровой виджет, или при выключении устройства, и в подобных ситуациях.

Объект `TapCancelEvent` содержит только `pointerId` предыдущего `TapDownEvent`, которое теперь отменяется. С отменой касания не связана никакая позиция.


### Демонстрация

Поиграйте с демонстрацией ниже, чтобы увидеть события касания в действии.

Синеватый прямоугольник в середине — это компонент с примесью `TapCallbacks`. При касании этого компонента в точках касания создаются круги. В частности, событие `onTapDown` начинает рисовать круг. Толщина круга будет пропорциональна длительности касания: после `onTapUp` толщина обводки круга перестанет расти. В момент срабатывания `onLongTapDown` появится тонкая белая полоска. Наконец, круг схлопнется и исчезнет, если вы вызовете событие `onTapCancel`, переместив палец.

## Примеси

В этом разделе более подробно описаны несколько примесей, необходимых для обработки событий касания.


### TapCallbacks

Примесь `TapCallbacks` может быть добавлена к любому `Component`, чтобы этот компонент начал получать события касания.

Эта примесь добавляет в компонент методы `onTapDown`, `onLongTapDown`, `onTapUp` и `onTapCancel`, которые по умолчанию ничего не делают, но могут быть переопределены для реализации реальной функциональности. Нет необходимости переопределять их все: например, можно переопределить только `onTapUp`, если вы хотите реагировать только на «настоящие» касания.

Ещё одна важная деталь: компонент будет получать только те события касания, которые происходят *внутри* этого компонента, что определяется функцией `containsLocalPoint()`. Широко используемый класс `PositionComponent` предоставляет такую реализацию на основе своего свойства `size`. Таким образом, если ваш компонент наследуется от `PositionComponent`, убедитесь, что правильно задали его размер. Если же ваш компонент наследуется от чистого `Component`, то метод `containsLocalPoint()` необходимо реализовать вручную.

Если ваш компонент является частью более крупной иерархии, он будет получать события касания только в том случае, если его родитель корректно реализовал `containsLocalPoint`.

```dart
class MyComponent extends Component with TapCallbacks {
  final _rect = const Rect.fromLTWH(0, 0, 100, 100);
  final _paint = Paint();
  bool _isPressed = false;

  @override
  bool containsLocalPoint(Vector2 point) => _rect.contains(point.toOffset());

  @override
  void onTapDown(TapDownEvent event) => _isPressed = true;

  @override
  void onTapUp(TapUpEvent event) => _isPressed = false;

  @override
  void onTapCancel(TapCancelEvent event) => _isPressed = false;

  @override
  void render(Canvas canvas) {
    _paint.color = _isPressed? Colors.red : Colors.white;
    canvas.drawRect(_rect, _paint);
  }
}
```


### SecondaryTapCallbacks

В дополнение к основным событиям касания (левая кнопка мыши на десктопе) Flame также поддерживает вторичные события касания (правая кнопка мыши на десктопе). Чтобы получать эти события, добавьте примесь `SecondaryTapCallbacks` к вашему `PositionComponent`.

```dart
class MyComponent extends PositionComponent with SecondaryTapCallbacks {
  @override
  void onSecondaryTapUp(SecondaryTapUpEvent event) {
    /// Выполнить действие
  }

  @override
  void onSecondaryTapCancel(SecondaryTapCancelEvent event) {
    /// Выполнить действие
  }

  @override
  void onSecondaryTapDown(SecondaryTapDownEvent event) {
    /// Выполнить действие
  }
```

Вы можете одновременно расширить `TapCallbacks` и `SecondaryTapCallbacks` в одном компоненте, чтобы получать как основные, так и вторичные события касания.


### TertiaryTapCallbacks

Flame также поддерживает третичные события касания (средняя кнопка мыши на десктопе). Чтобы получать эти события, добавьте примесь `TertiaryTapCallbacks` к вашему `PositionComponent`.

```dart
class MyComponent extends PositionComponent with TertiaryTapCallbacks {
  @override
  void onTertiaryTapUp(TertiaryTapUpEvent event) {
    /// Выполнить действие
  }

  @override
  void onTertiaryTapCancel(TertiaryTapCancelEvent event) {
    /// Выполнить действие
  }

  @override
  void onTertiaryTapDown(TertiaryTapDownEvent event) {
    /// Выполнить действие
  }
```

Вы можете комбинировать `TapCallbacks`, `SecondaryTapCallbacks` и `TertiaryTapCallbacks` в одном компоненте для независимого получения основных, вторичных и третичных событий касания.


### DoubleTapCallbacks

Flame также предлагает примесь `DoubleTapCallbacks` для получения событий двойного касания компонентом. Чтобы начать получать события двойного касания в компоненте, добавьте примесь `DoubleTapCallbacks` к вашему `PositionComponent`.

```dart
class MyComponent extends PositionComponent with DoubleTapCallbacks {
  @override
  void onDoubleTapUp(DoubleTapEvent event) {
    /// Выполнить действие
  }

  @override
  void onDoubleTapCancel(DoubleTapCancelEvent event) {
    /// Выполнить действие
  }

  @override
  void onDoubleTapDown(DoubleTapDownEvent event) {
    /// Выполнить действие
  }
```


## Миграция

Если у вас есть существующая игра, использующая примеси `Tappable`/`Draggable`, то в этом разделе описывается, как перейти на новый API, описанный в этом документе. Вот что нужно сделать:

Возьмите все компоненты, использующие эти примеси, и замените их на `TapCallbacks`/`DragCallbacks`. Методы `onTapDown`, `onTapUp`, `onTapCancel` и `onLongTapDown` потребуется адаптировать под новый API:

- Пара аргументов, например `(int pointerId, TapDownDetails details)`, была заменена одним объектом события `TapDownEvent event`.
- Возвращаемого значения больше нет, но если нужно, чтобы компонент пропускал касания к нижележащим компонентам, установите `event.continuePropagation` в true. Это необходимо только для событий `onTapDown`; все остальные события будут пропускаться автоматически.
- Если вашему компоненту нужно знать координаты точки касания, используйте `event.localPosition` вместо вычисления вручную. Также доступны свойства `event.canvasPosition` и `event.devicePosition`.
- Если компонент прикреплён к пользовательскому предку, убедитесь, что этот предок также имеет правильный размер или реализует `containsLocalPoint()`.
